# Heartbeat 心跳服务

## 概述

Heartbeat 服务是一个定期唤醒机制，让 Agent 在没有用户触发的情况下主动执行任务。Agent 会定期读取工作空间中的 `HEARTBEAT.md` 文件，并执行其中定义的任务。

## 配置

在 `config.json` 中启用心跳服务：

```json
{
  "heartbeat": {
    "enabled": true,
    "interval": 30
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | true | 是否启用心跳服务 |
| `interval` | int | 30 | 心跳间隔（分钟） |

## 工作原理

```
┌─────────────────────────────────────────────────┐
│              HeartbeatService                    │
│                                                  │
│   每 N 分钟（默认 30）                           │
│        ↓                                         │
│   读取 workspace/HEARTBEAT.md 文件              │
│        ↓                                         │
│   文件为空？ ──是──→ 跳过，等待下次心跳          │
│        │                                         │
│       否                                         │
│        ↓                                         │
│   唤醒 Agent 执行任务                            │
│        ↓                                         │
│   任务完成？──是──→ 回复 "HEARTBEAT_OK"          │
│        │                                         │
│       否                                         │
│        ↓                                         │
│   执行任务并记录结果                             │
└─────────────────────────────────────────────────┘
```

## 使用方法

1. 在工作空间目录（默认 `~/.lingguard/workspace/`）创建 `HEARTBEAT.md` 文件

2. 在文件中定义 Agent 需要定期检查的任务

3. 启动 gateway，心跳服务会自动运行

## 示例 HEARTBEAT.md

```markdown
# 心跳任务清单

## 定期检查项

1. 检查系统日志是否有错误
2. 检查是否有待处理的任务

## 如果发现问题

- 记录到 HISTORY.md
- 如果严重，发送通知到飞书群

## 无任务时

回复: HEARTBEAT_OK
```

## 响应格式

- **无任务需要处理**: `HEARTBEAT_OK` 或包含此标识的响应
- **有任务完成**: 描述执行的操作
- **需要人工介入**: 描述问题并请求帮助

## 使用场景

1. **定时监控** - 定期检查系统状态、日志文件
2. **数据同步** - 定期拉取最新数据或配置
3. **提醒服务** - 在特定时间主动发送提醒
4. **自动化运维** - 定期执行清理、备份任务
5. **社交互动** - 检查消息、回复评论（如 Moltbook 技能）

## 注意事项

- 心跳会占用 LLM API 调用次数
- 如果 HEARTBEAT.md 为空或不存在，心跳会被跳过
- 心跳使用独立的会话 ID (`heartbeat-main`)，不影响用户对话
- 心跳超时时间为 5 分钟
